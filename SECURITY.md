# Security policy

## Supported use

Pokala HealthOps is a local-first demonstration system. It is safe only for synthetic/demo data and small public reference samples.

## Prohibited data

Do not upload, paste, commit, ingest, or process:

- PHI
- employer data
- credentials
- secrets
- production EHR exports
- real patient records
- private operational screenshots
- private logs

## Security boundary

The public site is a documentation and evidence surface. It must not accept uploads. The API is intended for local demo use unless a separate deployment review is completed.

## Compliance boundary

This project is not a HIPAA certification, not a medical device, and not clinical decision support.

## Reporting issues

Open a GitHub issue for security concerns that do not include private data. Do not include secrets or PHI in an issue.
