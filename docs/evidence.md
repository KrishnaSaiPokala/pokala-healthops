# Evidence

Evidence is the proof layer for the incident demo. The project should not rely on screenshots or claims.

## Evidence files

| File | Purpose |
| --- | --- |
| `evidence/incident-report.json` | Static incident evidence committed for review. |
| `evidence/quality-checks.json` | Warehouse verification sample. |
| `evidence/dlq-before.json` | DLQ state before replay. |
| `evidence/dlq-after.json` | DLQ state after replay. |
| `evidence/audit-events.json` | Audit trail sample. |
| `evidence/metrics-sample.prom` | Prometheus sample output. |
| `evidence/replay-invariants.json` | Runtime invariant report generated by the CLI. |

## Runtime proof

Run:

```bash
python -m openhip.cli incident-demo
python -m openhip.cli replay-incident
python -m openhip.cli verify-warehouse
python -m openhip.cli verify-replay-invariants
python -m openhip.cli export-incident-report
```

The invariant command checks:

- incident is remediated
- open DLQ is zero
- replay failures are zero
- observations were not duplicated
- required audit actions exist
- warehouse checks pass

## Why this matters

A staff-level portfolio project should prove behavior with state, not copy. The evidence files make the core incident reviewable without trusting the homepage.
